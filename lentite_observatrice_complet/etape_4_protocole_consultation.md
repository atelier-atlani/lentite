# Étape 4 — Protocole de consultation et mise à jour mémoire
## Entité IA d'observation critique du réel

---

## DEUX TEMPS — AVANT ET APRÈS

La mémoire intervient à deux moments précis dans chaque analyse.
Elle n'intervient pas pendant — pour ne pas biaiser l'observation.

```
AVANT l'analyse → Consultation (orienter sans prédéterminer)
APRÈS l'analyse → Mise à jour (enrichir sans accumuler)
```

---

## TEMPS AVANT — CONSULTATION PRÉ-ANALYSE

### Étape 0.5 — Consultation mémoire (s'insère entre §0 et §1.1)

Avant de commencer la fiche d'observation, consulter les cinq
registres dans cet ordre :

**Consultation 1 — Registre 1 (analyses archivées)**

```
Existe-t-il un dossier archivé de type similaire ?
  Critères de similarité : type d'objet + type d'acteur
  + registre discursif + secteurs concernés

Si oui :
  → Lire les hypothèses retenues et les degrés de confiance
  → Lire "ce qui ferait changer l'analyse"
  → Utiliser comme hypothèse de départ à tester,
    pas comme conclusion à reproduire
```

**Consultation 2 — Registre 2 (erreurs)**

```
Ce type d'objet a-t-il déjà produit des erreurs récurrentes ?

Si oui :
  → Lire les conditions déclenchantes
  → Intégrer comme points de vigilance dans la fiche d'observation
  → Ne pas "corriger" préventivement — rester attentif
```

**Consultation 3 — Registre 3 (hypothèses ouvertes)**

```
Des hypothèses ouvertes sur des dossiers liés peuvent-elles
être testées par ce nouveau dossier ?

Si oui :
  → Signaler dans la fiche : "ce dossier peut apporter des
    indices sur l'hypothèse ouverte [ID]"
  → Ne pas orienter l'analyse vers cette hypothèse —
    seulement noter la possibilité
```

**Consultation 4 — Registre 4 (calibration méthodes)**

```
Ce type d'objet a-t-il une entrée dans la matrice ?

Si oui :
  → Appliquer la méthode principale recommandée
  → Ajouter les méthodes complémentaires
  → Être particulièrement attentif à l'angle souvent manqué
```

**Consultation 5 — Registre 5 (patterns)**

```
Des patterns sectoriels ou discursifs s'appliquent-ils à ce dossier ?

Si oui :
  → Formuler comme hypothèse préliminaire dans §0 :
    "Le pattern [ID] suggère que [formulation] —
     à tester, pas à conclure"
  → Degré de confiance initial : Faible
    (un pattern n'est pas une preuve sur un nouveau dossier)
```

### Résumé de la consultation

```
RÉSUMÉ CONSULTATION MÉMOIRE

Dossiers similaires trouvés : [liste ou "aucun"]
Erreurs à surveiller sur ce type : [liste ou "aucune connue"]
Hypothèses ouvertes potentiellement testables : [liste ou "aucune"]
Méthode recommandée par la calibration : [ou "pas de données"]
Patterns applicables : [liste ou "aucun"]

Impact sur la fiche d'observation :
[Ce que la consultation change dans §0 et §1]
```

## TEMPS APRÈS — MISE À JOUR MÉMOIRE

### Étape finale — Mise à jour (s'effectue après T8/synthèse finale)

**Mise à jour 1 — Registre 1 (archiver l'analyse)**

Créer une entrée pour ce dossier. Obligatoire.

**Mise à jour 2 — Registre 2 (erreurs)**

Pour chaque objection valide reçue des agents :
- Créer une entrée d'erreur
- Relier au type O/EO/JO/SO/DO correspondant
- Formuler la règle apprise
- Signaler si une modification du protocole est nécessaire

**Mise à jour 3 — Registre 3 (hypothèses)**

Pour chaque hypothèse ouverte existante :
- Ce dossier l'a-t-il renforcée, affaiblie, ou laissée stable ?
- Mettre à jour le statut

Pour chaque hypothèse formulée dans ce dossier qui reste ouverte :
- Créer une entrée dans le registre 3

**Mise à jour 4 — Registre 4 (calibration méthodes)**

Pour ce dossier :
- Quelle méthode a bien fonctionné ?
- Quelle méthode a manqué quelque chose ?
- Mettre à jour la matrice

**Mise à jour 5 — Registre 5 (patterns)**

Si un pattern transversal a été observé pour la première fois :
- Créer une entrée dans 5A, 5B ou 5C

Si un pattern existant a été confirmé sur un nouveau dossier :
- Ajouter le dossier à la liste + mettre à jour le degré de confiance

---

## RÈGLE DE CONSULTATION HONNÊTE

La consultation ne doit pas orienter l'analyse vers
une conclusion attendue. Elle oriente l'attention,
pas la conclusion.

Si un pattern suggère "ce secteur bénéficie toujours de X",
ce n'est pas une permission de conclure à X sur le nouveau dossier.
C'est une permission de chercher X plus attentivement.

La différence entre orienter l'attention et orienter la conclusion
est la différence entre apprendre et confirmer ses biais.

## EXEMPLE — CONSULTATION AVANT LOI IMMIGRATION

Si on analyse la loi immigration après avoir archivé la Loi Climat :

**Consultation 1** : dossier Loi Climat → similitudes ?
Type d'objet : décision formelle → similaire
Registre discursif Loi Climat : moral/transition
Registre Loi Immigration : souverain/sécuritaire → différent
→ Hypothèses Climate non directement transférables
→ Mais : pattern du filtrage (mesures qui coûtent aux organisés
  abandonnées, mesures qui profitent aux organisés maintenues)
  est potentiellement testable

**Consultation 2** : erreurs sur décisions formelles ?
→ Erreur récurrente : distribution sociale des coûts non tracée
→ Point de vigilance sur immigration : qui supporte les coûts
  du régime d'immigration tel qu'il est (travailleurs dans
  les secteurs concernés, bénéficiaires de services publics
  dans les zones de concentration)

**Consultation 4** : méthode pour décision formelle ?
→ Matrice indique : chaîne décisionnelle + alternatives écartées
   + attention aux effets sectoriels (angle souvent manqué)
→ Sur immigration : chercher les secteurs économiques qui
  bénéficient du status quo ou du changement proposé

**Consultation 5** : patterns applicables ?
→ Pattern 5B (secteurs qui bénéficient d'une précarité de statut)
  → agriculture, BTP, services à la personne
→ Formuler comme hypothèse préliminaire Faible :
  "Ces secteurs peuvent bénéficier du régime de précarité
   induite par le statut migratoire — à chercher, pas à conclure"

## CRITÈRE DE MATURITÉ DU SYSTÈME MÉMOIRE

Le système mémoire est mature quand :

1. La consultation pré-analyse dure moins de 10 minutes
   et produit des points de vigilance utiles sur 80%+ des dossiers

2. Les mises à jour post-analyse prennent moins de 20 minutes
   et produisent au moins une entrée nouvelle sur chaque registre

3. Au moins un pattern transversal est détecté tous les
   5 dossiers traités

4. Les erreurs récurrentes diminuent entre les dossiers 1-5
   et les dossiers 6-10

5. La calibration des méthodes est consultée avant chaque analyse
   et produit des décisions méthodologiques documentées
